Policy Statement
Purpose

The backup of mission critical information is often the last line of defence in the event of accidental or malicious loss or the unintended modification of application data and infrastructure configurations. The purpose of this policy is to set out a baseline requirement for the backup of xnPOS application and configuration data.

 

This information must be backed up on a regular basis, protected from unauthorised access or modification during storage, and available for recovery in a timely manner. As backup media may contain sensitive information in high-volumes, the backup media must be protected from physical and environmental damage.
Scope

This product policy applies to XPS staff, its customers, and any third-parties with access to XPS hosted business solutions. This policy applies to all hosted applications that contain information that might impact customers in the event data was lost.

1.1.2 Communications

The backup requirements for information systems and data must be documented and communicated to implementation and support teams for inclusion within operational procedures before systems entering production.

1.2 Backup verification

1.2.1 Verification

A sample of backup jobs must be verified as part of the process to maintain the integrity of the information being backed up, in a manner commensurate with the reliability of the backup media.

XPS makes regular backup recovery tests in order to verify that online backups can be

recovered and that consistency exists between file data and database information.

1.2.2 Failure

Backup failure reports must be produced, reviewed and acted upon within a reasonable time-frame to ensure successful completion.

1.3 Protection of backups and backup media

1.3.1 Encryption

Backup media is regarded with an equivalent security classification level as the source information system. Sensitive data must be appropriately encrypted (e.g., at the database or file level) when stored on backup media.

1.3.2 Secure Access

Access to backup media must be restricted to authorised personnel only.

1.4 Retention and disposal of backups and backup media

1.4.1 Retention

Backup media must be retained in line with the IT recovery, data retention and record management requirements where applicable.

1.4 Backup media locations

1.4.1 Physical location

Backup media must be stored in a safe and secure physical location to ensure that media is protected from unauthorised access, modification or destruction. This includes:

  1. Off-site in relation to the XPS data centre and stored at a location with strict physical security in place.
  2. In a temperature controlled environment employing fire prevention suppression mechanisms

1.4.2 Data replication

XPS uses disk backups to help protect against the loss of Customer production data. All backups are

stored at a geographically separate location from the primary production environment. This is accomplished through data replication via SQL Server AlwaysOn Availability Groups, Veritas Backup Exec, and tiered storage (HDD and SSD).
